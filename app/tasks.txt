1) Функция для получение результата логических условий, которая вызывает функцию исполнения результата
    а) Предусмотреть вложенность условий в поле result
    б) Предусмотреть последовательно несколько условий
    в) Если в logic есть только операции без условий вернуть сразу их
2) Функция для исполнения результатов условий - операций, через класс в services
3) API для создания стратегий и привязывания ее к продукту
4) Создание celery таска при сохранении стратегии
    а) При сохранении стратегии, взять активную и с самой высокой приоритетностью стратегию от продукта сохраненной стратегии
    б) Вызвать первую функцию передавая полученную стратегию как celery task с периодичностью 30 минут
5) В функции которая испоняет результат нужно записывать совершенные операции в модельку Стратегии
    Например:
        "operations": {
                            "variable": "медианная цена конкурентов",
                            "do": "="
                      }
        Мы записываем в journals - [{
            "operation_{datetime}":  {product.current_price_before_discount} = {медианная цена конкурентов} = {product.new_price_before_discount}
        }]
6) Доработать парсинг exel файлов
    а) Сделать модель xlsx файла (id, product, original_file, changed_file), привязать к продукту
    б) При сохранении этой модели вызывать парсинг этого файла и запись в бд
    в) В конце функции исполнения celery task'а вызывать функцию записи новых значений в таблицу и сохранять новый файл в поле changed_file
